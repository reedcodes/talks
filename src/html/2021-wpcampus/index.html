./package.json

{
  "name": "sample-kss",
  "version": "1.0.0",
  "description": "Sample KSS",
  "devDependencies": {
    "sass": "1.35.*",
    "kss": "3.0.*"
  },
  "scripts": {
    "build:kss": "kss --config kss-config.json",
    "build:css": "sass --no-source-map src:dist/css"
  }
}
- include the scripts to build css and kss - no files, so scripts do nothing right now

./kss-config.json

{
  "title": "Sample KSS",
  "source": "src/",
  "destination": "styleguide/",
  "builder": "builder/twig/",
  "css": "../dist/css/styles.css"
}
- Configure the KSS .json File - The source, destination, and builder are relative to the kss-config.json file - The css path is relative to the generated style guide files

./kss-config.json

{
  "title": "Sample KSS",
  "source": "src/",
  "destination": "styleguide/",
  "builder": "builder/",
  "css": "../dist/css/styles.css",
  "namespace": [
    "color:src/color",
    "text:src/text",
    "button:src/button",
    "card:src/card",
    "layout:src/layout"
  ]
}
because we'll reference Twig from pattern library, custom blocks, custom themes, let's give our patterns a namespace

Add `./src/styles.scss` now so we have something to build from later. 
// Brand Colors
//
// Colors:
// --darkgray: rgb(37, 41, 53) - Dark Gray
// --purple: rgb(125, 59, 106) - Purple
// --cobalt: rgb(0, 121, 140) - Cobalt
// --crimson: rgb(208, 55, 17) - Crimson
// --goldenrod: rgb(246, 174, 45) - Goldenrod
// --icewhite: rgb(248, 245, 242) - Ice White
//
// Weight: 1
//
// Styleguide: Color

:root {
  --darkgray: rgb(37, 41, 53);
  --purple: rgb(125, 59, 106);
  --cobalt: rgb(0, 121, 140);
  --crimson: rgb(208, 55, 17);
  --goldenrod: rgb(246, 174, 45);
  --icewhite: rgb(248, 245, 242);
}
- First: name of the pattern - brand colors - Second: list of the colors - name, code, description - Third: weight of the page - Next: section of the styleguide, used for creating hierarchy and page URLs - After comments: regular Sass or CSS declarations
